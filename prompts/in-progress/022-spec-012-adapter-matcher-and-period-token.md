<summary>
- The Go side now understands a `weekday` value that is a single day OR a list of days, and accepts both long names (`Monday`) and short names (`Mon`), normalizing them to canonical `time.Weekday` values at parse time.
- A `Schedule` CR that targets `[Mon, Wed, Fri]` produces ONE in-memory task definition carrying the set `{Monday, Wednesday, Friday}` — downstream code never branches on long-vs-short form.
- The day-matcher fires a `Weekday` task on any day whose weekday is in the set; over one ISO week `[Mon, Wed, Fri]` yields exactly three task files, each with its own per-day period token (`-mon`, `-wed`, `-fri`) and its own UUID5.
- An existing single-day CR (`weekday: Monday`) produces a one-element set and byte-identical output (same UUID5, same period token, same Kafka message) — no vault file regenerates.
- Mixed-form lists (`[Mon, Wednesday, Fri]`) parse to the same definition as the all-long-form equivalent.
- An unknown day string encountered at parse time is wrapped, the offending CR is dropped for the cycle, the error is logged, and every other CR continues.
- Existing `Daily`/`Weekly`/`Monthly`/`Quarterly`/`Yearly` behavior, period tokens, and the `/trigger` contract are untouched.
</summary>

### LOAD-BEARING DESIGN DECISION (resolved here; do not reopen)

The spec lets the agent choose how the day-set flows into per-day period tokens. The chosen, single design:

1. `TaskDefinition` carries `Weekdays []time.Weekday` — a canonical SET (replacing the singular `Weekday time.Weekday` field). For `RecurrenceWeekday` it is non-empty; for every other kind it is nil/empty and ignored.
2. The matcher (`filterInventoryByDate`) fires a `RecurrenceWeekday` def when `date.Time().Weekday()` is a member of `def.Weekdays`.
3. The period token for `RecurrenceWeekday` uses **the firing date's weekday** (`date.Time().Weekday()`), NOT a stored single value. On a firing day the date's weekday is guaranteed in the set, so a single-day `[Monday]` CR firing on a Monday produces token `YYYYWww-mon` — byte-identical to the pre-spec single-`time.Monday` output. A multi-day `[Mon,Wed,Fri]` CR naturally produces `-mon` on Monday, `-wed` on Wednesday, `-fri` on Friday as the tick advances day by day — three distinct tokens, three UUID5s, three task files per ISO week.
4. Because the token now derives from the firing date, the `PeriodTokenBuilder` no longer reads `def.Weekday`/`def.Weekdays` for the weekday suffix. The matcher guarantees the date is a firing day; defensively, if `date.Time().Weekday()` is NOT in `def.Weekdays` for a `RecurrenceWeekday` def, `Build` returns a wrapped error (this path is unreachable through the tick, but guards direct misuse).

This is the only design. Do not implement a "stored single weekday for the token" variant.

```go
// WeekdayList is the spec.schedule.weekday value. The CRD wire shape is
// a single day string OR a non-empty list of day strings (long form
// Monday..Sunday or short form Mon..Sun, freely mixed) — see the OpenAPI
// OneOf in pkg/k8s_connector_schema.go. WeekdayList normalizes both wire
// shapes to a []string on decode: a bare string decodes to a one-element
// slice. Canonicalization of short->long form and the mapping to
// time.Weekday happens Go-side in the store adapter, not here — this type
// only unifies the two JSON shapes.
type WeekdayList []string

// UnmarshalJSON accepts either a JSON string ("Monday") or a JSON array
// (["Mon","Wed"]) and stores the result as a []string. A null or absent
// value yields a nil slice.
func (w *WeekdayList) UnmarshalJSON(data []byte) error {
	trimmed := bytes.TrimSpace(data)
	if len(trimmed) == 0 || string(trimmed) == "null" {
		*w = nil
		return nil
	}
	if trimmed[0] == '[' {
		var list []string
		if err := json.Unmarshal(trimmed, &list); err != nil {
			return err
		}
		*w = list
		return nil
	}
	var single string
	if err := json.Unmarshal(trimmed, &single); err != nil {
		return err
	}
	*w = []string{single}
	return nil
}

// MarshalJSON renders a one-element list back as a bare string (so an
// applied single-day CR round-trips to the same wire form) and a
// multi-element list as a JSON array. An empty list renders as null.
func (w WeekdayList) MarshalJSON() ([]byte, error) {
	switch len(w) {
	case 0:
		return []byte("null"), nil
	case 1:
		return json.Marshal(w[0])
	default:
		return json.Marshal([]string(w))
	}
}
```
