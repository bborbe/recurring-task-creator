<summary>
- The Schedule CRD gains an optional `skipAutoCleanup` boolean field (default off) so an operator can mark a schedule as "preserve every missed instance" instead of auto-closing prior ones.
- Applying a CR with `skipAutoCleanup: true`, `false`, or the field absent all succeed; applying it with a non-bool string is rejected at the API server.
- The internal task-definition model carries the flag as a plain bool, defaulting to false when the CR omits it.
- Every generated vault task now carries an `audit_style` frontmatter key mirroring the flag, so an operator can grep the vault and confirm which tasks are the audit-style ones.
- No new package, no cron, no git-rest in this prompt — this is purely the wire-format delta plus the publisher's new stamped key.
- Existing schedules and their task identifiers are unchanged.
</summary>
